## 意思決定

### 川ヒートマップをOSM GeoJSONに切り替え

- **決定**: river-tilesタイル方式を廃止しOverpass API GeoJSONで描画
- **理由**: タイル方式はCarto暗色マップのピクセル解析に依存しており不安定。OSMの実データをL.geoJSON()で描画する方が精度・保守性ともに優る
- **実装**: fetch_osm_rivers.py → osm_rivers_tokyo.geojson（1,418フィーチャー / 東京23区）

### GeoJSONサイズ対応
- 初期取得3,758フィーチャー/2.8MB → river/canal/大水体のみに絞り1,418件/1.4MB
- nginx gzip有効化（application/json追加）で転送量を約30%に圧縮
- ブラウザキャッシュで2回目以降はゼロオーバーヘッド
