## 導入・実装（詳細）

* OSM Overpass API GeoJSON方式によるヒートマップ川表示
  - waterway=river/canal + natural=water + waterway=riverbank
  - 3766フィーチャー / 3.0MB
  - L.geoJSON()でLeaflet描画
  - 川名ホバーツールチップ付き
  - nginxでgzip圧縮
* 国土数値情報W09湖沼データ統合（8件のみ：狭山湖・多摩湖等）
* tokyo_rivers_combined.geojsonとして統合保存

## 廃止・変更（詳細）

  - 理由：タイル座標のズレが全zoomレベルで発生し解決不可能と判断
  - 事前生成324MB/29025ファイルを全削除
  - CartoDB DarkタイルのピクセルRGB解析は不安定
* river-tilesフォルダ削除（VPS容量+324MB解放）
