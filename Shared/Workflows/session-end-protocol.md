# セッション終了プロトコル

> 目的: 次回Claudeが5秒以内に現在地を復元できる状態を作る。
> 全文保存ではなく「意思決定の記録」と「現在地マーカー」を残す。

---

## 実行手順（全4ステップ）

---

### STEP 1 — current-task.md を更新（現在地マーカー）

**役割:** Claudeの現在地。次回セッション開始時に最初に読むファイル。

**更新条件:**

| 状況 | 更新する内容 |
|---|---|
| ステップが進んだ | `step` フィールドと `[ ]`→`[x]` チェックリスト |
| 未決定事項が解決した | Input テーブルの `未定` → 決定内容 |
| 新しいブロッカーが発生 | `blocker` セクションに追記 |
| 中断・割り込みが発生 | `checkpoint` に中断地点を記録 |
| Inboxからアイデアを取り込んだ | Notes に記録 |

**`checkpoint` フィールドのルール:**
- 「どのステップのどこまで終わったか」を1行で書く
- 例: `STEP 2完了・STEP 3のFlux生成プロンプト作成中・感情アーク設計済み`
- 次回Claudeはここから再開する

**更新しない条件:**
- セッション内容がcurrent-taskと無関係な作業（Vault整備など）
- 対象プロジェクトに変化がなかった

---

### STEP 2 — PROJECT_STATUS.md を更新（全体MAP）

**役割:** 全プロジェクトの俯瞰マップ。詳細はcurrent-taskに委ねる。

**更新条件:**

| フィールド | 更新タイミング |
|---|---|
| `current_status` | ステータスが変わったとき（active→review 等） |
| `next_action` | 毎回更新（次セッションの起点として） |
| `blocker` | ブロッカーの発生・解消時 |
| `review_waiting` | レビュー提出・承認時 |
| `latest_output` | 成果物（画像・動画・LP等）が出たとき |
| `updated_at` | 毎回更新 |

**更新しない条件:**
- `current_goal` — ゴールが変わったときのみ更新（毎回書き換えない）
- `priority` — tagishiが指示したときのみ変更

**ルール:**
- next_action は「動詞 + 対象 + 目的」の1文で書く
- 例: `○ タスクXをtagishiに確認してSTEP 2へ進む`
- 例: `× 確認する`（不明瞭すぎる）

---

### STEP 3 — Log を書く（意思決定履歴）

**役割:** 全文記録ではなく「何を決めたか・なぜ・何を却下したか」の記録。
ファイル名: `Logs/YYYY/MM/YYYY-MM-DD-session-[番号].md`

**書く内容（必須）:**

```
## 意思決定ログ
- [決定した内容] → [理由]
- [却下した選択肢] → [却下理由]
```

**書かない内容（禁止）:**
- 実行した操作の列挙（作成した・更新した etc.）
- ファイル変更の全リスト
- Claude自身の思考プロセスの全文

**Blocker 保存ルール:**
- ブロッカーは「問い」の形で書く
- 例: `○ tagishiに「ターゲットは20代女性か30代男性か」を確認する必要がある`
- 例: `× ターゲット未定`（問いの形になっていない）

**保存条件（いずれかに該当する場合のみ作成）:**
1. 重要な意思決定があった
2. ブロッカーが発生または解消した
3. プロジェクトのステップが進んだ
4. Vault構造に変更があった

**保存しない場合:** 雑談・確認のみ・変更なしのセッション

---

### STEP 4 — SESSION END SUMMARY を出力

セッション終了時、以下のテンプレートを必ず出力する。
これがセッション記録の唯一のユーザー向け出力。

```
---
## Session End Summary — [日付] #[セッション番号]

**対象プロジェクト:** [name]
**ステータス:** [idle|active|review|blocked]

### 今回やったこと（3行以内）
-
-
-

### 意思決定
| 決定 | 理由 |
|---|---|
| — | — |

### 現在地（checkpoint）
[current-task.md の checkpoint 内容をそのままコピー]

### 次回の first action
> [1文・動詞から始める]

### ブロッカー
- [なし または 具体的な問いの形で]

---
```

---

## 保存の優先順位

```
速度 優先 ──────────────────────────── 精度 優先

current-task.md   PROJECT_STATUS.md   Log
（必ず更新）        （必ず更新）          （条件付き）
```

current-task と PROJECT_STATUS は**毎回更新**。
Log は**条件を満たした場合のみ**作成する。

- [ ] PROJECT_STATUS.md の `updated_at` を今日の日付にした
